AIBUS

AI-Powered Business Utility Suite

An AI-powered Business Intelligence platform integrating forecasting, anomaly detection, customer segmentation, AI-generated insights, and interactive dashboards. Engineered a modular analytics pipeline supporting datasets with over 1M+ records, featuring automated preprocessing, schema detection, feature engineering, and exportable PDF reports.

GitHub Repository

The Problem

Small and medium enterprises (SMEs) face a challenge: they aggregate large quantities of transaction and log data but lack the dedicated analytics budgets of enterprise companies. Without robust toolsets, these businesses rely on manual spreadsheets to calculate historical summaries.

This results in delays, human errors, and a complete lack of forward-looking predictions (such as forecasting demand spikes or identifying rising cost anomalies before they impact the bottom line).

The Solution

This platform bridges that gap by deploying a lightweight, open-source BI stack. By engineering an automated Python data ingestion pipeline, raw logs are cleaned, loaded, and processed automatically.

The interactive Streamlit front-end renders historical sales trends, runs customer segmentation algorithms automatically to cluster buyer profiles, and fits predictive time-series models to output 30-day forecast projections.

System Architecture

Key Features

  • AI-Powered BI: Integrates forecasting, anomaly detection, customer segmentation, AI-generated insights, and interactive dashboards.
  • Modular Analytics Pipeline: Supports datasets with over 1M+ records, featuring automated preprocessing, schema detection, and feature engineering.
  • Advanced Forecasting: Fits predictive time-series models (Prophet) to output business projections.
  • Exportable PDF Reports: Generates automated PDF reports summarizing data insights and visualizations.

Tech Stack

Python Streamlit Pandas Plotly Scikit-learn Prophet

Key Challenge

Ensuring the predictive models performed fast enough for real-time dashboard adjustments. Initially, running validation loops on model updates blocked the interface. Resolved this by separating statistical model fitting tasks from the UI thread and caching computed model matrices using Streamlit's internal cache state mechanism.

Outcome & Impact

Developed as a modular analytics pipeline, the system successfully supports datasets with over 1M+ records, featuring automated preprocessing, schema detection, feature engineering, and exportable PDF reports, while providing seamless real-time forecasts and segmentations.
