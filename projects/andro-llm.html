ANDRO-LLM

Offline Local Conversational AI for Mobile Devices

A native Android implementation of highly optimized, quantized large language models (LLMs) executing local, offline inference via a modern, clean Flutter UI dashboard.

Timeline
4 Months (2025)
Role
Mobile & Core Developer
Languages
C++, Dart, Kotlin
Backend
llama.cpp Engine
GitHub Repository

The Problem

Contemporary LLM systems rely heavily on remote cloud services (such as OpenAI, Anthropic, or Gemini API endpoints). This requires constant active network connectivity, incurs network billing, and creates significant data privacy concerns.

Recruiters and enterprise users processing sensitive personal summaries or offline fields are hesitant to pass records to external servers, creating a major need for local conversational frameworks.

The Solution

ANDRO-LLM bypasses external cloud dependencies by compiling llama.cpp natively for Android targets. By compiling raw C++ libraries, model weights can be read directly from device flash memory.

A responsive Flutter frontend communicates with the underlying compiled native libraries via Dart FFI channels, offering smooth, zero-latency chats and real-time generation previews.

Process Architecture

Key Features

  • Offline Conversational Engine: Runs complete text processing locally without internet.
  • Quantized Model Execution: Loads 4-bit integer quantized (GGUF) parameters, minimizing RAM usage to fit standard mobile memory ceilings.
  • Native Compilation: Compiles C++ source utilizing ARM Neon SIMD intrinsics for optimized CPU calculations.
  • Dart FFI: Fast data sharing across the managed Flutter runtime and native memory space.

Tech Stack

Flutter Dart C++ llama.cpp Android NDK CMake

Key Challenge

Handling garbage collection during massive text generation events. Passing large chunks of string allocations from native memory to Dart's managed garbage collector caused visual frame drops. Resolved this by establishing pre-allocated pointer buffers and streaming token indices instead of raw texts.

Outcome & Impact

ANDRO-LLM showcases how modern mobile hardware can run local generative models without relying on remote API subscriptions. On testing mid-tier devices (e.g. Snapdragon 778G), the system outputs 12-14 tokens per second utilizing a 3B parameter model, maintaining memory under 2.5GB.
